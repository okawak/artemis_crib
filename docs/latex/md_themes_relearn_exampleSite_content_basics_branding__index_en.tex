The Relearn theme provides configuration options to change your site\textquotesingle{}s colors, favicon and logo. This allows you to easily align your site visuals to your desired style. Most of these options are exposed through so called color variants.

A color variant lets you customize various visual effects of your site like almost any color, used fonts, color schemes of print, syntax highligtning, Mermaid and the Open\+API shortcode, etc. It contains of a CSS file and optional configuration options in your {\ttfamily hugo.\+toml}.

The Relearn theme ships with a wide set of different color variants. You can use them as-\/is, copy them over and use them as a starting point for your customizations or just create completely new variants unique to your site. The \href{basics/generator}{\texttt{ interactive variant generator}} may help you with this task.

Once configured in your {\ttfamily hugo.\+toml}, you can select them with the variant selector at the bottom of the menu.\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_theme-variant}{}\doxysubsection{Change the Variant (\+Simple)}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_theme-variant}
\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md726}{}\doxysubsubsection{Single Variant}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md726}
Set the {\ttfamily theme\+Variant} value to the name of your theme file. That\textquotesingle{}s it! Your site will be displayed in this variant only.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant = \char`\"{}relearn-\/light\char`\"{} \{\{$<$ /multiconfig $>$\}\}

\{\{\% notice note \%\}\} Your theme variant file must reside in your site\textquotesingle{}s {\ttfamily assets/css} directory and the file name must start with {\ttfamily theme-\/} and end in {\ttfamily .css}. In the above example, the path of your theme file must be {\ttfamily assets/css/theme-\/relearn-\/light.\+css}.

If you want to make changes to a shipped color variant, create a copy in your site\textquotesingle{}s {\ttfamily assets/css} directory. Don\textquotesingle{}t edit the file in the theme\textquotesingle{}s directory! \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md727}{}\doxysubsubsection{Multiple Variants}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md727}
You can also set multiple variants. In this case, the first variant is the default chosen on first view and a variant selector will be shown in the menu footer if the array contains more than one entry.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant = \mbox{[} \char`\"{}relearn-\/light\char`\"{}, \char`\"{}relearn-\/dark\char`\"{} \mbox{]} \{\{$<$ /multiconfig $>$\}\}

\{\{\% notice tip \%\}\} The theme provides an advanced configuration mode, combining the functionality for multiple variants with the below possibilities of adjusting to your OS settings and syntax highlighting and even more!

Although all options documented here are still working, the advanced configuration options are the recommended way to configure your color variants. \href{\#theme-variant-advanced}{\texttt{ See below}}. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md728}{}\doxysubsection{Adjust to OS Settings}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md728}
You can also cause the site to adjust to your OS settings for light/dark mode. Just set the {\ttfamily theme\+Variant} to {\ttfamily auto} to become an auto mode variant. That\textquotesingle{}s it.

You can use the {\ttfamily auto} value with the single or multiple variants option. If you are using multiple variants, you can drop {\ttfamily auto} at any position in the option\textquotesingle{}s array, but usually it makes sense to set it in the first position and make it the default.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant = \mbox{[} \char`\"{}auto\char`\"{}, \char`\"{}red\char`\"{} \mbox{]} \{\{$<$ /multiconfig $>$\}\}

If you don\textquotesingle{}t configure anything else, the theme will default to use {\ttfamily relearn-\/light} for light mode and {\ttfamily relearn-\/dark} for dark mode. These defaults are overwritten by the first two non-\/auto options of your {\ttfamily theme\+Variant} option if present.

In the above example, you would end with {\ttfamily red} for light mode and the default of {\ttfamily relearn-\/dark} for dark mode.

If you don\textquotesingle{}t like that behavior, you can explicitly set {\ttfamily theme\+Variant\+Auto}. The first entry in the array is the color variant for light mode, the second for dark mode.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant\+Auto = \mbox{[} \char`\"{}learn\char`\"{}, \char`\"{}neon\char`\"{} \mbox{]} \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md729}{}\doxysubsection{Change the Favicon}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md729}
If your favicon is a SVG, PNG or ICO, just drop your image in your site\textquotesingle{}s {\ttfamily static/images/} directory and name it {\ttfamily favicon.\+svg}, {\ttfamily favicon.\+png} or {\ttfamily favicon.\+ico} respectively.

If you want to adjust your favicon according to your OS settings for light/dark mode, add the image files {\ttfamily static/images/favicon-\/light.\+svg} and {\ttfamily static/images/favicon-\/dark.\+svg} to your site\textquotesingle{}s directory, respectively, corresponding to your file format. In case some of the files are missing, the theme falls back to {\ttfamily favicon.\+svg} for each missing file. All supplied favicons must be of the same file format.

If no favicon file is found, the theme will lookup the alternative filename {\ttfamily logo} in the same location and will repeat the search for the list of supported file types.

If you need to change this default behavior, create a new file {\ttfamily layouts/partials/favicon.\+html} in your site\textquotesingle{}s directory and write something like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<link rel="{}icon"{} href="{}/images/favicon.bmp"{} type="{}image/bmp"{}>}

\end{DoxyCode}
\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md730}{}\doxysubsection{Change the Logo}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md730}
Create a new file in {\ttfamily layouts/partials/logo.\+html} of your site. Then write any HTML you want. You could use an {\ttfamily img} HTML tag and reference an image created under the {\itshape static} folder, or you could paste a SVG definition!

\{\{\% notice note \%\}\} The size of the logo will adapt automatically. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md731}{}\doxysubsection{Syntax Highlighting}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_autotoc_md731}
If you want to switch the syntax highlighting theme together with your color variant, you need to configure your installation \href{https://gohugo.io/content-management/syntax-highlighting/}{\texttt{ according to Hugo\textquotesingle{}s documentation}} and provide a syntax highlighting stylesheet file.

You can use a one of the shipped stylesheet files or use Hugo to generate a file for you. The file must be written to {\ttfamily assets/css/chroma-\/$<$NAME$>$.css}. To use it with your color variant you have to define {\ttfamily -\/-\/CODE-\/theme\+: $<$NAME$>$} in the color variant stylesheet file.

For an example, take a look into \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/assets/css/theme-relearn-light.css}{\texttt{ {\ttfamily theme-\/relearn-\/light.\+css}}} and \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/exampleSite/config/_default/hugo.toml}{\texttt{ {\ttfamily hugo.\+toml}}} of the example\+Site.\hypertarget{md_themes_relearn_exampleSite_content_basics_branding__index_en_theme-variant-advanced}{}\doxysubsection{Change the Variant (\+Advanced)}\label{md_themes_relearn_exampleSite_content_basics_branding__index_en_theme-variant-advanced}
The theme offers a new way to configure theme variants and all of the aspects above inside of a single configuration item. This comes with some features previously unsupported.

Like with the \href{\#multiple-variants}{\texttt{ multiple variants}} option, you are defining your theme variants in an array but now {\itshape not by simple strings} {\bfseries{but in a table with suboptions}}.

Again, in this case, the first variant is the default chosen on first view and a variant selector will be shown in the menu footer if the array contains more than one entry.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant = \mbox{[} \char`\"{}relearn-\/light\char`\"{}, \char`\"{}relearn-\/dark\char`\"{} \mbox{]} \{\{$<$ /multiconfig $>$\}\}

you now write it that way\+:

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} \mbox{[}\mbox{[}params.\+theme\+Variant\mbox{]}\mbox{]} identifier = \char`\"{}relearn-\/light\char`\"{} \mbox{[}\mbox{[}params.\+theme\+Variant\mbox{]}\mbox{]} identifier = \char`\"{}relearn-\/dark\char`\"{} \{\{$<$ /multiconfig $>$\}\}

In case you like a shipped variant but only want to tweak some aspects, you have two choices\+:


\begin{DoxyEnumerate}
\item Copy and change

You can copy the shipped variant file from the theme\textquotesingle{}s {\ttfamily assets/css} directory to the site\textquotesingle{}s {\ttfamily assets/css} directory and either store it with the same name or give it a new name. Edit the settings and save the new file. Afterwards you can use it in your {\ttfamily hugo.\+toml} by the chosen name.
\item Create and import

You can create a new variant file in the site\textquotesingle{}s {\ttfamily assets/css} directory and give it a new name. Import the shipped variant, add the settings you want to change and save the new file. Afterwards you can use it in your {\ttfamily hugo.\+toml} by the chosen name.

For example, you want to use the {\ttfamily relearn-\/light} variant but want to change the syntax highlighting schema to the one used in the {\ttfamily neon} variant. For that, create a new {\ttfamily assets/css/theme-\/my-\/branding.\+css} in your site\textquotesingle{}s directory and add the following lines\+:

\`{}\`{}\`{}\`{}css \{title=\char`\"{}assets/css/theme-\/my-\/branding.\+css\char`\"{}\} @import \char`\"{}theme-\/relearn-\/light.\+css\char`\"{};

\+:root \{ --CODE-\/theme\+: neon; /$\ast$ name of the chroma stylesheet file $\ast$/ --CODE-\/\+BLOCK-\/color\+: rgba( 226, 228, 229, 1 ); /$\ast$ fallback color for code text $\ast$/ --CODE-\/\+BLOCK-\/\+BG-\/color\+: rgba( 40, 42, 54, 1 ); /$\ast$ fallback color for code background $\ast$/ \} \`{}\`{}\`{}\`{}

Afterwards put this in your {\ttfamily hugo.\+toml} to use your new variant\+:

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} theme\+Variant = \char`\"{}my-\/branding\char`\"{} \{\{$<$ /multiconfig $>$\}\}

In comparison to {\itshape copy and change}, this has the advantage that you profit from any adjustments to the {\ttfamily relearn-\/light} variant but keep your modifications. 
\end{DoxyEnumerate}
