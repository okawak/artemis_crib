A new release can happen at any time from the {\ttfamily main} branch of the \href{https://github.com/McShelby/hugo-theme-relearn}{\texttt{ Git\+Hub project}} without further acknowledgment. This makes it necessary that, every pushed set of changesets into the {\ttfamily main} branch {\bfseries{must}} be self-\/contained and correct, resulting in a releasable version.

Stay simple for the user by focusing on the mantra \char`\"{}convention over configuration\char`\"{}.

At installation the site should work reasonable without (m)any configuration.

Stay close to the Hugo way.

Don\textquotesingle{}t use npm or any preprocessing, our contributors may not be front-\/end developers.

Document new features in the example\+Site. This also contains entries to the \href{basics/migration}{\texttt{ What\textquotesingle{}s new}} page.

Don\textquotesingle{}t break existing features if you don\textquotesingle{}t have to.

Remove reported issue from the browser\textquotesingle{}s console.

Check for unnecessary whitespace and correct indention of your resulting HTML.\hypertarget{md_themes_relearn_exampleSite_content_dev_contributing__index_en_autotoc_md959}{}\doxysubsection{Conventional Commits}\label{md_themes_relearn_exampleSite_content_dev_contributing__index_en_autotoc_md959}
