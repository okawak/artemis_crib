The {\ttfamily highlight} shortcode renders your code with a syntax highlighter.

\{\{$<$ highlight line\+Nos=\char`\"{}true\char`\"{} type=\char`\"{}py\char`\"{} wrap=\char`\"{}true\char`\"{} title=\char`\"{}python\char`\"{} $>$\}\} print(\char`\"{}\+Hello World!\char`\"{}) \{\{$<$ /highlight $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_shortcodes_highlight_en_autotoc_md1047}{}\doxysubsection{Usage}\label{md_themes_relearn_exampleSite_content_shortcodes_highlight_en_autotoc_md1047}
This shortcode is fully compatible with Hugo\textquotesingle{}s \href{https://gohugo.io/content-management/syntax-highlighting/\#highlight-shortcode}{\texttt{ {\ttfamily highlight} shortcode}} but {\bfseries{offers some extensions}}.

It is called interchangeably in the same way as Hugo\textquotesingle{}s own shortcode providing positional parameter or by simply using codefences.

You are free to also call this shortcode from your own partials. In this case it resembles Hugo\textquotesingle{}s \href{https://gohugo.io/functions/highlight/}{\texttt{ {\ttfamily highlight} function}} syntax if you call this shortcode as a partial using compatibility syntax.

While the examples are using shortcodes with named parameter it is recommended to use codefences instead. This is because more and more other software supports codefences (eg. Git\+Hub) and so your markdown becomes more portable.
