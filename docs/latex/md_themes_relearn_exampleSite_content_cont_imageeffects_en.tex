The theme supports non-\/standard \href{cont/markdown\#image-effects}{\texttt{ image effects}}.

\tabulinesep=1mm
\begin{longtabu}spread 0pt [c]{*{2}{|X[-1]}|}
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Name   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Description    }\\\cline{1-2}
\endfirsthead
\hline
\endfoot
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Name   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Description    }\\\cline{1-2}
\endhead
border   &Draws a light thin border around the image    \\\cline{1-2}
lazy   &Lets the image be lazy loaded    \\\cline{1-2}
lightbox   &The image will be clickable to show it enlarged    \\\cline{1-2}
shadow   &Draws a shadow around the image to make it appear hovered/glowing   \\\cline{1-2}
\end{longtabu}


As \href{cont/markdown\#image-effects}{\texttt{ described}}, you can add this to the URL query parameter, but this may be cumbersome to be done consistently for the whole page.

Instead, you can configure the defaults in your {\ttfamily hugo.\+toml} as well as overriding these default in the pages frontmatter.

Explicitly set URL query parameter will override the defaults in effect for a page.

Without any settings in your {\ttfamily hugo.\+toml} this defaults to

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} \mbox{[}params.\+image\+Effects\mbox{]} border = false lazy = true lightbox = true shadow = false \{\{$<$ /multiconfig $>$\}\}

This can be overridden in a pages frontmatter by eg.

\{\{$<$ multiconfig fm=true $>$\}\} \mbox{[}image\+Effects\mbox{]} border = true \{\{$<$ /multiconfig $>$\}\}

Or by explicitly override settings by URL query parameter


\begin{DoxyCode}{0}
\DoxyCodeLine{![Minion](https://octodex.github.com/images/minion.png?lightbox=false\&bg-\/white=true)}

\end{DoxyCode}


The settings applied to the above image would be

\{\{$<$ multiconfig $>$\}\} border = true lazy = true lightbox = false shadow = false bg-\/white = true \{\{$<$ /multiconfig $>$\}\}

This ends up in the following HTML where the parameter are converted to CSS classes.


\begin{DoxyCode}{0}
\DoxyCodeLine{<img src="{}https://octodex.github.com/images/minion.png?lightbox=false\&bg-\/white=true"{} loading="{}lazy"{} alt="{}Minion"{} class="{}bg-\/white border lazy nolightbox noshadow"{}>}

\end{DoxyCode}
\hypertarget{md_themes_relearn_exampleSite_content_cont_imageeffects_en_autotoc_md897}{}\doxysubsection{Extending}\label{md_themes_relearn_exampleSite_content_cont_imageeffects_en_autotoc_md897}
As you can see in the above example, the {\ttfamily bg-\/white} parameter is not initially supported in the themes default settings. Nevertheless you are free to define arbitrary parameter by just adding them to the URL query parameter or set them in your {\ttfamily hugo.\+toml} or pages frontmatter.

\{\{\% notice note \%\}\} If no extended parameter like {\ttfamily bg-\/white} in the example is set on the URL, a {\ttfamily class=\char`\"{}nobg-\/white\char`\"{}} in the HTML will only be generated if a default value was set in the {\ttfamily hugo.\+toml} or pages frontmatter. \{\{\% /notice \%\}\}
