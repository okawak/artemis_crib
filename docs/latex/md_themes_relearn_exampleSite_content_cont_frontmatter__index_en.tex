\hypertarget{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md898}{}\doxysubsubsection{Ordering Sibling Menu/\+Page Entries}\label{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md898}
Hugo provides a \href{https://gohugo.io/content/ordering/}{\texttt{ flexible way}} to handle order for your pages.

The simplest way is to set {\ttfamily weight} parameter to a number.

\{\{$<$ multiconfig fm=true $>$\}\} title = \char`\"{}\+My page\char`\"{} weight = 5 \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md899}{}\doxysubsubsection{Using a Custom Title for Menu Entries}\label{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md899}
By default, the Relearn theme will use a page\textquotesingle{}s {\ttfamily title} attribute for the menu item.

But a page\textquotesingle{}s title has to be descriptive on its own while the menu is a hierarchy. Hugo adds the {\ttfamily link\+Title} parameter for that purpose\+:

For example (for a page named {\ttfamily content/install/linux.\+md})\+:

\{\{$<$ multiconfig fm=true $>$\}\} title = \char`\"{}\+Install on Linux\char`\"{} link\+Title = \char`\"{}\+Linux\char`\"{} \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md900}{}\doxysubsubsection{Override Expand State Rules for Menu Entries}\label{md_themes_relearn_exampleSite_content_cont_frontmatter__index_en_autotoc_md900}
You can change how the theme expands menu entries on the side of the content with the {\ttfamily alwaysopen} setting on a per page basis. If {\ttfamily alwaysopen=false} for any given entry, its children will not be shown in the menu as long as it is not necessary for the sake of navigation.

The theme generates the menu based on the following rules\+:


\begin{DoxyItemize}
\item all parent entries of the active page including their siblings are shown regardless of any settings
\item immediate children entries of the active page are shown regardless of any settings
\item if not overridden, all other first level entries behave like they would have been given {\ttfamily alwaysopen=false}
\item if not overridden, all other entries of levels besides the first behave like they would have been given {\ttfamily alwaysopen=true}
\item all visible entries show their immediate children entries if {\ttfamily alwaysopen=true}; this proceeds recursively
\item all remaining entries are not shown
\end{DoxyItemize}

You may want to structure your pages in a hierarchical way but don\textquotesingle{}t want to generate pages for those sections? The theme got you covered.
