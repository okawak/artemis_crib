By default, shortcuts are preceded by a title. This title can be disabled by setting {\ttfamily disable\+Shortcuts\+Title=true}. However, if you want to keep the title but change its value, it can be overridden by changing your local i18n translation string configuration.

For example, in your local {\ttfamily i18n/en.\+toml} file, add the following content


\begin{DoxyCode}{0}
\DoxyCodeLine{[Shortcuts-\/Title]}
\DoxyCodeLine{other = "{}<Your value>"{}}

\end{DoxyCode}

If you have shortcuts to pages inside of your project and you don\textquotesingle{}t want them to show up in page menu section, you have two choices\+:


\begin{DoxyEnumerate}
\item Make the page file for the shortcut a \href{https://gohugo.io/content-management/page-bundles/\#headless-bundle}{\texttt{ headless branch bundle}} (contained in its own subdirectory and called {\ttfamily \+\_\+index.\+md}) and add the following frontmatter configuration to the file (see example\+Site\textquotesingle{}s {\ttfamily \mbox{\hyperlink{showcase_2__index_8en_8md}{content/showcase/\+\_\+index.\+en.\+md}}}). This causes its content to {\bfseries{not}} be ontained in the sitemap.

\{\{$<$ multiconfig fm=true $>$\}\} title = \char`\"{}\+Showcase\char`\"{} \mbox{[}\+\_\+build\mbox{]} render = \char`\"{}always\char`\"{} list = \char`\"{}never\char`\"{} publish\+Resources = true \{\{$<$ /multiconfig $>$\}\}
\item Store the page file for the shortcut below a parent headless branch bundle and add the following frontmatter to he {\bfseries{parent}} (see example\+Site\textquotesingle{}s {\ttfamily \mbox{\hyperlink{more_2__index_8en_8md}{content/more/\+\_\+index.\+en.\+md}}}). {\bfseries{Don\textquotesingle{}t give this page a {\ttfamily title}}} as this will cause it to be shown in the breadcrumbs -\/ a thing you most likely don\textquotesingle{}t want.

\{\{$<$ multiconfig fm=true $>$\}\} \mbox{[}\+\_\+build\mbox{]} render = \char`\"{}never\char`\"{} list = \char`\"{}never\char`\"{} publish\+Resources = false \{\{$<$ /multiconfig $>$\}\}

In this case, the file itself can be a branch bundle, leaf bundle or simple page (see example\+Site\textquotesingle{}s {\ttfamily content/more/credits.\+en.\+md}). This causes its content to be contained in the sitemap.

\{\{$<$ multiconfig fm=true $>$\}\} title = \char`\"{}\+Credits\char`\"{} \{\{$<$ /multiconfig $>$\}\} 
\end{DoxyEnumerate}
