The Relearn theme is fully compatible with Hugo multilingual mode.


\begin{DoxyItemize}
\item Available languages\+: Arabic, Simplified Chinese, Traditional Chinese, Czech, Dutch, English, Finnish, French, German, Hindi, Hungarian, Indonesian, Italian, Japanese, Korean, Polish, Portuguese, Romanian, Russian, Spanish, Swahili, Turkish, Vietnamese. Feel free to contribute!
\item Full support for languages written right to left
\item Automatic menu generation from multilingual content
\item In-\/browser language switching
\end{DoxyItemize}

\hypertarget{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md902}{}\doxysubsection{Basic configuration}\label{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md902}
After learning \href{https://gohugo.io/content-management/multilingual}{\texttt{ how Hugo handle multilingual websites}}, define your languages in your {\ttfamily hugo.\+toml} file.

For example with current English and Piratized English website.

\{\{\% notice note \%\}\} Make sure your default language is defined as the first one in the {\ttfamily \mbox{[}languages\mbox{]}} array, as the theme needs to make assumptions on it \{\{\% /notice \%\}\}

\{\{$<$ multiconfig file=hugo $>$\}\} default\+Content\+Language = \char`\"{}en\char`\"{}

\mbox{[}languages\mbox{]} \mbox{[}languages.\+en\mbox{]} title = \char`\"{}\+Hugo Relearn Theme\char`\"{} weight = 1 language\+Name = \char`\"{}\+English\char`\"{}

\mbox{[}languages.\+pir\mbox{]} title = \char`\"{}\+Cap\textquotesingle{}n Hugo Relearrrn Theme\char`\"{} weight = 2 language\+Name = \char`\"{}\+Arrr! Pirrrates\char`\"{} \{\{$<$ /multiconfig $>$\}\}

Then, for each new page, append the {\itshape id} of the language to the file.


\begin{DoxyItemize}
\item Single file {\ttfamily my-\/page.\+md} is split in two files\+:
\begin{DoxyItemize}
\item in English\+: {\ttfamily my-\/page.\+md}
\item in Piratized English\+: {\ttfamily my-\/page.\+pir.\+md}
\end{DoxyItemize}
\item Single file {\ttfamily \+\_\+index.\+md} is split in two files\+:
\begin{DoxyItemize}
\item in English\+: {\ttfamily \+\_\+index.\+md}
\item in Piratized English\+: {\ttfamily \+\_\+index.\+pir.\+md}
\end{DoxyItemize}
\end{DoxyItemize}

\{\{\% notice info \%\}\} Be aware that only translated pages are displayed in menu. It\textquotesingle{}s not replaced with default language content. \{\{\% /notice \%\}\}

\{\{\% notice tip \%\}\} Use \href{https://gohugo.io/content-management/multilingual/\#translate-your-content}{\texttt{ slug}} frontmatter parameter to translate urls too. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md903}{}\doxysubsection{Search}\label{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md903}
In case each page\textquotesingle{}s content is written in one single language only, the above configuration will already configure the site\textquotesingle{}s search functionality correctly.

\{\{\% notice warning \%\}\} Although the theme supports a wide variety of supported languages, the site\textquotesingle{}s search via the \href{https://lunrjs.com}{\texttt{ Lunr}} search library does not. You\textquotesingle{}ll see error reports in your browsers console log for each unsupported language. Currently unsupported are\+:


\begin{DoxyItemize}
\item Czech
\item Indonesian
\item Polish
\item Swahili \{\{\% /notice \%\}\}
\end{DoxyItemize}\hypertarget{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md904}{}\doxysubsubsection{Search with mixed language support}\label{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md904}
In case your page\textquotesingle{}s content contains text in multiple languages (e.\+g. you are writing a Russian documentation for your english API), you can add those languages to your {\ttfamily hugo.\+toml} to broaden search.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}params\mbox{]} additional\+Content\+Language = \mbox{[} \char`\"{}en\char`\"{} \mbox{]} \{\{$<$ /multiconfig $>$\}\}

As this is an array, you can add multiple additional languages.

\{\{\% notice note \%\}\} Keep in mind that the language code required here, is the base language code. E.\+g. if you have additional content in {\ttfamily zh-\/\+CN}, you have to add just {\ttfamily zh} to this parameter. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md905}{}\doxysubsection{Overwrite translation strings}\label{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md905}
Translations strings are used for common default values used in the theme ({\itshape Edit} button, {\itshape Search placeholder} and so on). Translations are available in English and Piratized English but you may use another language or want to override default values.

To override these values, create a new file in your local i18n folder {\ttfamily i18n/$<$idlanguage$>$.toml} and inspire yourself from the theme {\ttfamily themes/hugo-\/theme-\/relearn/i18n/en.\+toml}\hypertarget{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md906}{}\doxysubsection{Disable language switching}\label{md_themes_relearn_exampleSite_content_cont_i18n__index_en_autotoc_md906}
Switching the language in the browser is a great feature, but for some reasons you may want to disable it.

Just set {\ttfamily disable\+Language\+Switching\+Button=true} in your {\ttfamily hugo.\+toml}
