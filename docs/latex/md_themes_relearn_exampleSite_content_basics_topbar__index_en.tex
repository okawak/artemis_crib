The theme comes with a reasonably configured topbar.



Nevertheless, your requirements may differ from this configuration. Luckily the theme got you covered as the themebar, its buttons and the functionality behind these buttons is fully configurable by you.

\{\{\% notice tip \%\}\} All mentioned file names below can be clicked and show you the implementation for a better understanding. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md872}{}\doxysubsection{Areas}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md872}
The default configuration comes with three predefined areas that may contain an arbitrary set of buttons.




\begin{DoxyItemize}
\item \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/area/start.html}{\texttt{ {\bfseries{start}}}}\+: shown between menu and breadcrumb
\item \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/area/end.html}{\texttt{ {\bfseries{end}}}}\+: shown on the opposite breadcrumb side in comparison to the {\itshape start} area
\item \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/area/more.html}{\texttt{ {\bfseries{more}}}}\+: shown when pressing the \{\{\% button style=\char`\"{}transparent\char`\"{} icon=\char`\"{}ellipsis-\/v\char`\"{} \%\}\}\{\{\% /button \%\}\} {\itshape more} button in the topbar
\end{DoxyItemize}

While you can not add additional areas in the topbar, you are free to configure additional buttons that behave like the {\itshape more} button, providing further user defined areas.\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md873}{}\doxysubsection{Buttons}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md873}
The theme ships with the following predefined buttons (from left to right in the screenshot)


\begin{DoxyItemize}
\item \{\{\% button style=\char`\"{}transparent\char`\"{} icon=\char`\"{}bars\char`\"{} \%\}\}\{\{\% /button \%\}\} \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/sidebar.html}{\texttt{ {\bfseries{sidebar}}}}\+: opens the sidebar flyout if in mobile layout
\item \{\{\% button style=\char`\"{}transparent\char`\"{} icon=\char`\"{}list-\/alt\char`\"{} \%\}\}\{\{\% /button \%\}\} \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/toc.html}{\texttt{ {\bfseries{toc}}}}\+: opens the table of contents in an overlay

\item \{\{\% button style=\char`\"{}transparent\char`\"{} icon=\char`\"{}print\char`\"{} \%\}\}\{\{\% /button \%\}\} \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/print.html}{\texttt{ {\bfseries{print}}}}\+: browses to the chapters printable page if \href{basics/customization\#activate-print-support}{\texttt{ print support}} was activated

\end{DoxyItemize}

Not all buttons are displayed at every given time. This is configurable (see below if interested).\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md874}{}\doxysubsection{Redefining Areas}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md874}
Each predefined area and button comes in their own file. By that it is easy for you to overwrite an area file in your installation reusing only the buttons you like.

Eg. you can redefine the predefined {\itshape end} area by adding the file \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/area/end.html}{\texttt{ {\ttfamily layouts/partials/topbar/area/end.\+html}}} in your installation (not in the theme itself) to remove all but the {\itshape more} button.

The below example sets an explicit value for the {\ttfamily onempty} parameter, overriding the specific default value for this button (these defaults vary depending on the button). The parameter causes the {\itshape more} button to always be displayed instead of hiding once its content is empty.

\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md875}{}\doxysubsection{Defining own Buttons}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md875}
\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md876}{}\doxysubsubsection{Button Types}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md876}
The theme distinguishes between two types of buttons\+:


\begin{DoxyItemize}
\item \href{\#button}{\texttt{ {\bfseries{button}}}}\+: a clickable button that either browses to another site, triggers a user defined script or opens an overlay containing user defined content
\item \href{\#area-button}{\texttt{ {\bfseries{area-\/button}}}}\+: the template for the \{\{\% button style=\char`\"{}transparent\char`\"{} icon=\char`\"{}ellipsis-\/v\char`\"{} \%\}\}\{\{\% /button \%\}\} {\itshape more} button, to define your own area overlay buttons
\end{DoxyItemize}\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md877}{}\doxysubsubsection{Button Parameter}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md877}
\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md878}{}\doxyparagraph{Screen Widths and Actions}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md878}
Depending on the screen width you can configure how the button should behave. Screen width is divided into three classes\+:


\begin{DoxyItemize}
\item {\bfseries{s}}\+: (controlled by the {\ttfamily onwidths} parameter) mobile layout where the menu sidebar is hidden
\item {\bfseries{m}}\+: (controlled by the {\ttfamily onwidthm} parameter) desktop layout with visible sidebar while the content area width still resizes
\item {\bfseries{l}}\+: (controlled by the {\ttfamily onwidthl} parameter) desktop layout with visible sidebar once the content area reached its maximum width
\end{DoxyItemize}

For each width class, you can configure one of the following actions\+:


\begin{DoxyItemize}
\item {\ttfamily show}\+: the button is displayed in its given area
\item {\ttfamily hide}\+: the button is removed
\item {\ttfamily area-\/\+XXX}\+: the button is moved from its given area into the area {\ttfamily XXX}; eg. this is used to move buttons to the {\itshape more} area overlay in the mobile layout
\end{DoxyItemize}\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md879}{}\doxyparagraph{Hiding and Disabling Stuff}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md879}
While hiding a button depending on the screen size can be configured with the above described {\itshape hide} action, you may want to hide the button on certain other conditions as well.

For example, the {\itshape print} button in its default configuration should only be displayed if print support was configured. This is done in your button template by checking the conditions first before displaying the button (see \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/print.html}{\texttt{ {\ttfamily layouts/partials/topbar/button/print.\+html}}}).

Another preferred condition for hiding a button is, if the displayed overlay is empty. This is the case for the {\itshape toc} (see \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/toc.html}{\texttt{ {\ttfamily layouts/partials/topbar/button/toc.\+html}}}) as well as the {\itshape more} button (see \href{https://github.com/McShelby/hugo-theme-relearn/blob/main/layouts/partials/topbar/button/more.html}{\texttt{ {\ttfamily layouts/partials/topbar/button/more.\+html}}}) and controlled by the parameter {\ttfamily onempty}.

This parameter can have one of the following values\+:


\begin{DoxyItemize}
\item {\ttfamily disable}\+: the button displayed in disabled state if the overlay is empty
\item {\ttfamily hide}\+: the button is removed if the overlay is empty
\end{DoxyItemize}

If you want to disable a button containing {\itshape no overlay}, this can be achieved by an empty {\ttfamily href} parameter. An example can be seen in the {\itshape prev} button (see {\ttfamily layouts/partials/topbar/button/prev.\+html}) where the URL for the previous site may be empty.\hypertarget{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md880}{}\doxysubsection{Reference}\label{md_themes_relearn_exampleSite_content_basics_topbar__index_en_autotoc_md880}
