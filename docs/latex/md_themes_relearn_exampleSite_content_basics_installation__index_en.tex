The following steps are here to help you initialize your new website. If you don\textquotesingle{}t know Hugo at all, we strongly suggest you learn more about it by following this \href{https://gohugo.io/overview/quickstart/}{\texttt{ great documentation for beginners}}.

\{\{\% notice tip \%\}\} The following tutorial leads you through the steps of creating a first, minimal new site.

You don\textquotesingle{}t need to edit any files besides your {\ttfamily hugo.\+toml} and only need to execute the commands in the given order. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md753}{}\doxysubsection{Create your Project}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md753}
Hugo provides the {\ttfamily new} command to create a new website\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo new site my-\/new-\/site}

\end{DoxyCode}


After that change into the directory\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cd my-\/new-\/site}

\end{DoxyCode}


Every upcoming command will be executed from inside your new site\textquotesingle{}s root.\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md754}{}\doxysubsection{Install the Theme}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md754}
\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md755}{}\doxysubsubsection{Downloading as Archive}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md755}
You can \href{https://github.com/McShelby/hugo-theme-relearn/archive/main.zip}{\texttt{ download the theme as .zip archive}} and extract its content into them {\ttfamily themes/hugo-\/theme-\/relearn} directory.

Afterwards add this at the end of your {\ttfamily hugo.\+toml}.

\{\{$<$ multiconfig file=hugo $>$\}\} theme = \char`\"{}hugo-\/theme-\/relearn\char`\"{} \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md756}{}\doxysubsubsection{Using Hugo\textquotesingle{}s Module System}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md756}
You can install the Relearn theme by following \href{https://gohugo.io/hugo-modules/use-modules/\#use-a-module-for-a-theme}{\texttt{ the standard documentation}} using Hugo\textquotesingle{}s module system\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo mod init example.com}

\end{DoxyCode}


Afterwards add this at the end of your {\ttfamily hugo.\+toml}.

\{\{$<$ multiconfig file=hugo $>$\}\} \mbox{[}module\mbox{]} \mbox{[}\mbox{[}module.\+imports\mbox{]}\mbox{]} path = \textquotesingle{}github.\+com/\+Mc\+Shelby/hugo-\/theme-\/relearn\textquotesingle{} \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md757}{}\doxysubsubsection{Using Git Submodules}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md757}
If you plan to store your project in a git repository you can create one with\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git init}

\end{DoxyCode}


Now add the theme as a submodule by\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git submodule add https://github.com/McShelby/hugo-\/theme-\/relearn.git themes/hugo-\/theme-\/relearn}

\end{DoxyCode}


Afterwards add this at the end of your {\ttfamily hugo.\+toml}.

\{\{$<$ multiconfig file=hugo $>$\}\} theme = \char`\"{}hugo-\/theme-\/relearn\char`\"{} \{\{$<$ /multiconfig $>$\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md758}{}\doxysubsection{Create your Home Page}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md758}
If you don\textquotesingle{}t create a home page, yet, the theme will generate a placeholder text with instructions on how to proceed.

Start your journey by creating a home page\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo new -\/-\/kind home \_index.md}

\end{DoxyCode}


The newly created home page {\ttfamily \mbox{\hyperlink{__index_8md}{content/\+\_\+index.\+md}}} is empty and you obviously should add some meaningful content.\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md759}{}\doxysubsection{Create your First Chapter Page}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md759}
Chapters are meant to be top level pages that contain other child pages. They have a special layout style and often just contain the {\itshape title} and a {\itshape brief abstract} of the section.

Now create your first chapter page with the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo new -\/-\/kind chapter basics/\_index.md}

\end{DoxyCode}


When opening the newly created file {\ttfamily content/basics/\+\_\+index.\+md}, you should see the {\ttfamily weight} frontmatter with a number. This will be used to generate the subtitle of the chapter page, and should be set to a consecutive value starting at {\ttfamily 1} for each chapter level.\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md760}{}\doxysubsection{Create your First Content Pages}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md760}
Then create content pages inside the previously created chapter. Here are three ways to create content in the chapter\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo new basics/first-\/content/\_index.md}
\DoxyCodeLine{hugo new basics/second-\/content/index.md}
\DoxyCodeLine{hugo new basics/third-\/content.md}

\end{DoxyCode}


Feel free to edit those files by adding some sample content and replacing the {\ttfamily title} value in the beginning of the files.

\{\{\% notice note \%\}\} Please note that Hugo overrides the default archetype template coming with this theme when using {\ttfamily hugo new site my-\/new-\/site}. To actually see your page later, you have to remove the {\ttfamily draft=true} from the page\textquotesingle{}s frontmatter. \{\{\% /notice \%\}\}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md761}{}\doxysubsection{Testing your Website Locally}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md761}
Launch your new web site by using the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo serve}

\end{DoxyCode}


Go to \href{http://localhost:1313}{\texttt{ {\ttfamily http\+://localhost\+:1313}}} in your browser.

You should notice a few things\+:


\begin{DoxyEnumerate}
\item The home page contains your provided text.
\item You have the menu {\bfseries{Basics}} in the sidebar. Clicking on it reveals three submenus with names equal to the {\ttfamily title} properties in the previously created content pages.
\item While you are running {\ttfamily hugo serve} your page refreshes automatically when you change a content page. Neat!
\end{DoxyEnumerate}\hypertarget{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md762}{}\doxysubsection{Build and Deploy your Website}\label{md_themes_relearn_exampleSite_content_basics_installation__index_en_autotoc_md762}
When your site is ready to be deployed, run the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{hugo}

\end{DoxyCode}


A {\ttfamily public} directory will be generated, containing all content and assets for your web site.

It now can be deployed to any web server by simply uploading its contents or you can check out one of \href{https://gohugo.io/hosting-and-deployment/}{\texttt{ Hugo\textquotesingle{}s many other deployment options}}.

\{\{\% notice note \%\}\} If you are storing your web site in git, commit all but the {\ttfamily public} directory. \{\{\% /notice \%\}\}
