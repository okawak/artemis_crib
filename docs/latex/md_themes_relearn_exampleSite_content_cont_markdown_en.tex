Let\textquotesingle{}s face it\+: Writing content for the web is tiresome. WYSIWYG editors help alleviate this task, but they generally result in horrible code, or worse yet, ugly web pages.

{\bfseries{Markdown}} is a better way to write {\bfseries{HTML}}, without all the complexities and ugliness that usually accompanies it.

Some of the key benefits are\+:


\begin{DoxyEnumerate}
\item Markdown is simple to learn, with minimal extra characters so it\textquotesingle{}s also quicker to write content.
\item Less chance of errors when writing in Markdown.
\item Produces valid HTML output.
\item Keeps the content and the visual display separate, so you cannot mess up the look of your site.
\item Write in any text editor or Markdown application you like.
\item Markdown is a joy to use!
\end{DoxyEnumerate}

John Gruber, the author of Markdown, puts it like this\+:

\begin{quote}
The overriding design goal for Markdown’s formatting syntax is to make it as readable as possible. The idea is that a Markdown-\/formatted document should be publishable as-\/is, as plain text, without looking like it’s been marked up with tags or formatting instructions. While Markdown’s syntax has been influenced by several existing text-\/to-\/\+HTML filters, the single biggest source of inspiration for Markdown’s syntax is the format of plain text email. $<$cite$>$John Gruber$<$/cite$>$ \end{quote}
