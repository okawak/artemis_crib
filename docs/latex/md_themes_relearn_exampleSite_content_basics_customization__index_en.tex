\hypertarget{md_themes_relearn_exampleSite_content_basics_customization__index_en_autotoc_md752}{}\doxysubsection{Partials}\label{md_themes_relearn_exampleSite_content_basics_customization__index_en_autotoc_md752}
The Relearn theme has been built to be as configurable as possible by defining multiple \href{https://gohugo.io/templates/partials/}{\texttt{ partials}}

In {\ttfamily themes/hugo-\/theme-\/relearn/layouts/partials/}, you will find all the partials defined for this theme. If you need to overwrite something, don\textquotesingle{}t change the code directly. Instead \href{https://gohugo.io/themes/customizing/}{\texttt{ follow this page}}. You\textquotesingle{}d create a new partial in the {\ttfamily layouts/partials} folder of your local project. This partial will have the priority.

This theme defines the following partials \+:


\begin{DoxyItemize}
\item {\ttfamily header.\+html}\+: the header of the page. See \href{\#output-formats}{\texttt{ output-\/formats}}
\item {\ttfamily footer.\+html}\+: the footer of the page. See \href{\#output-formats}{\texttt{ output-\/formats}}
\item {\ttfamily body.\+html}\+: the body of the page. The body may contain of one or many articles. See \href{\#output-formats}{\texttt{ output-\/formats}}
\item {\ttfamily article.\+html}\+: the output for a single article, can contain elements around your content. See \href{\#output-formats}{\texttt{ output-\/formats}}
\item {\ttfamily menu.\+html}\+: left menu. {\itshape Not meant to be overwritten}
\item {\ttfamily search.\+html}\+: search box. {\itshape Not meant to be overwritten}
\item {\ttfamily custom-\/header.\+html}\+: custom headers in page. Meant to be overwritten when adding CSS imports. Don\textquotesingle{}t forget to include {\ttfamily style} HTML tag directive in your file.
\item {\ttfamily custom-\/footer.\+html}\+: custom footer in page. Meant to be overwritten when adding Java\+Script. Don\textquotesingle{}t forget to include {\ttfamily javascript} HTML tag directive in your file.
\item {\ttfamily favicon.\+html}\+: the favicon
\item {\ttfamily heading.\+html}\+: side-\/wide configuration to change the pages title headings.
\item {\ttfamily heading-\/pre.\+html}\+: side-\/wide configuration to prepend to pages title headings. If you override this, it is your responsibility to take the page\textquotesingle{}s {\ttfamily heading\+Pre} setting into account.
\item {\ttfamily heading-\/post.\+html}\+: side-\/wide configuration to append to pages title headings. If you override this, it is your responsibility to take the page\textquotesingle{}s {\ttfamily heading\+Post} setting into account.
\item {\ttfamily logo.\+html}\+: the logo, on top left hand corner
\item {\ttfamily meta.\+html}\+: HTML meta tags, if you want to change default behavior
\item {\ttfamily menu-\/pre.\+html}\+: side-\/wide configuration to prepend to menu items. If you override this, it is your responsibility to take the page\textquotesingle{}s {\ttfamily menu\+Pre} setting into account.
\item {\ttfamily menu-\/post.\+html}\+: side-\/wide configuration to append to menu items. If you override this, it is your responsibility to take the page\textquotesingle{}s {\ttfamily menu\+Post} setting into account.
\item {\ttfamily menu-\/footer.\+html}\+: footer of the left menu
\item {\ttfamily toc.\+html}\+: table of contents
\item {\ttfamily content.\+html}\+: the content page itself. This can be overridden if you want to display page\textquotesingle{}s meta data above or below the content.
\item {\ttfamily content-\/header.\+html}\+: header above the title, has a default implementation but you can overwrite it if you don\textquotesingle{}t like it.
\item {\ttfamily content-\/footer.\+html}\+: footer below the content, has a default implementation but you can overwrite it if you don\textquotesingle{}t like it. 
\end{DoxyItemize}
